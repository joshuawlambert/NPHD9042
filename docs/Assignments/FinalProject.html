Final Project Details and Guidelines

This academic project is a team effort involving a group of students, ideally with three members, unless specified otherwise by Dr. Lambert.

The main goal of the Final Project is to assess your understanding of Structural Equation Modeling (SEM). The project has three parts: a project proposal, a presentation, and a comprehensive paper, following the standard structure used in academic research, including Title page, Abstract, Introduction, Methods, Results, Discussion, and References.

The project proposal should be concise, focusing on the key ideas and excluding Results and Discussion. It lays the groundwork for the subsequent presentation. The presentation builds on the proposal by adding the omitted Results and Discussion sections, along with References. Feedback received during the presentation should be taken into account when finalizing the project paper.

The project paper is the most substantial part, requiring at least five pages. It involves a thorough exploration of the subject, with in-depth discussions tailored to the reviewer’s interests.

These components together simulate the academic research process, allowing participants to comprehensively utilize SEM through collaboration and scholarly investigation.

For the project, choose one dataset from the three provided below. Remember SEM consists of six steps:

  1. Choose Dataset (from list below).
  2. Model Specification: develop a theoretical model based on your groups research question or hypothesis.
  3. Model Estimation: use R to estimate the parameters of the model that best fit the data.
  4. Model Fit Evaluation: Comparative Fit Index (CFI), Root Mean Square Error of Approximation (RMSEA).
  5. Model Modification: modify the model by adding or removing paths, allowing latent variables to correlate, or including additional observed variables.
  6. Interpretation and Inference: Once your model is checked in step 5, interpret the estimated path coefficients to understand the relationships between variables and draw conclusions based on the findings.

Project Datasets

  1. Thomsen2.sav – R-ready with the haven package
  2. JobSatisfaction2.jmp – legacy JMP file; use a CSV/R-ready export before analyzing in R
  3. OnlineConsumerData2.jmp – legacy JMP file; use a CSV/R-ready export before analyzing in R

For more information about these datasets please see this document.

R Starter Code

The code below uses Thomsen2.sav, which can be read directly in R. Use it as a template for your own SEM model specification.

load_packages <- function(pkgs) {
  missing <- pkgs[!vapply(pkgs, requireNamespace, logical(1), quietly = TRUE)]
  if (length(missing) > 0) {
    install.packages(missing, repos = "https://cloud.r-project.org")
  }
  invisible(lapply(pkgs, library, character.only = TRUE))
}

assignment_data <- function(filename) {
  candidates <- c(filename, file.path("Assignments", filename), file.path("..", filename))
  hit <- candidates[file.exists(candidates)]
  if (length(hit) == 0) stop("Could not find ", filename)
  normalizePath(hit[1], winslash = "/")
}

load_packages(c("haven", "lavaan", "semPlot"))

thomsen <- haven::read_sav(assignment_data("Thomsen2.sav"))
thomsen <- as.data.frame(lapply(thomsen, as.numeric))

# Example SEM template. Update the latent variables and paths so they match
# your group's research question and theory.
sem_model <- '
  ECQ =~ ecq1 + ecq2 + ecq3 + ecq4 + ecq5 + ecq6 + ecq7 +
         ecq8 + ecq9 + ecq10 + ecq11 + ecq12 + ecq13 + ecq14

  POMS =~ poms4d + poms8d + poms12d + poms14d +
          poms20d + poms23d + poms28d + poms33d

  POMS ~ ECQ + psqik1 + consult + telephone
'

sem_fit <- lavaan::sem(sem_model, data = thomsen, missing = "fiml")
summary(sem_fit, fit.measures = TRUE, standardized = TRUE, rsquare = TRUE)
parameterEstimates(sem_fit, standardized = TRUE)
semPlot::semPaths(sem_fit, what = "std", edge.label.cex = 0.7, layout = "tree")

    • Creativity: Use original and engaging methods to present your results. Utilize external resources and materials to enhance your presentation.
    • Q&A: Reserve 5-10 minutes for questions.
    • Peer Review: Classmates will critique each presentation based on a scoring rubric. These reviews will be averaged for the peer review grade.

Project Paper

  • Content:
    • Introduction: Provide a comprehensive introduction to the topic and explain why this research is necessary and interesting.

    • Discussion and Conclusion: Summarize the findings, discuss implications, and suggest future research directions. Address possible pitfalls in the study.
